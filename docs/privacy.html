Hindsight

Privacy policy

Last updated: 2026-05-25 · Applies to Hindsight Chrome extension v0.5.x and later.

Plain-English summary Hindsight stores your captures on your own device. There is no Hindsight server. The extension has no telemetry, no analytics SDKs, and makes zero outbound network requests until you explicitly click a "Send" button. Authorization tokens, cookies, passwords, credit cards and Turkish national IDs are masked at capture time — the original values are never written to your local storage in the first place.

1. What we collect

When Hindsight is installed and enabled, content scripts inside the pages you visit and a service worker observe browser activity. Captures are categorised by tier:

Category Default Detail
Network requests On Method, URL, headers, body, status, response. Sensitive headers/bodies masked at capture time (see §3).
Console messages On Level, message, stack trace.
Navigation events On From-URL, to-URL, transition type.
Clicks On (Tier 2) Element tag, accessible name, position.
Form input changes On (Tier 2) Field name and type. Password / credit-card fields always masked.
Screenshots on error On (Tier 3) JPEG of visible tab on capture; throttled to one per tab per 2 s.
Performance signals On (Tier 3) Long tasks > 100 ms, cumulative layout shift.
Cursor / scroll trail Off (Tier 4) Only while Recording Mode is explicitly active.
Periodic screenshots Off (Tier 4) 2-second JPEGs, recording mode only.

Each tier is toggleable in Settings → Capture (except Tier 1, the bare minimum for the extension to function).

2. Where it goes

Everything is stored in chrome.storage.local on your own device. There is no Hindsight server. The repository that builds the extension is public and every line of code that touches fetch() or chrome.runtime.sendMessage is verifiable.

The service worker makes zero outbound HTTP requests of its own. The only fetches are:

  • A configured Slack / Discord / Teams webhook URL when you click "Send" on a captured event;
  • The "↻ Replay this request" button in the side panel, which re-fires a specific captured request from the extension context (without your real cookies — see §6).

3. Sensitive data — capture-time masking

Sensitive values are masked at capture time, before the event is written to storage:

  • Built-in header masking: Authorization, Cookie, Set-Cookie, X-API-Key, X-Auth-Token, Proxy-Authorization.
  • Built-in body pattern masking: Turkish national ID (TCKN, with checksum), credit cards (Luhn-validated 13–19 digits).
  • Built-in form masking: <input type="password">, credit-card autocomplete fields, fields named password / secret / token / ssn / pin.
  • User-defined body patterns — any regex you add in Settings → Privacy.

Matching values are replaced with the literal string ***MASKED*** before persistence. The original is never written down, never embedded in the screenshot, never copied to the HAR export, never included in any share payload. Mask cannot be reversed — there is nothing to reverse to.

You can disable individual rules in Settings → Privacy if you need raw values in your own captures. The disable is per-rule, per-future-capture, and the Settings page warns explicitly that captures recorded with a rule disabled will store the matched value verbatim on your machine.

4. The 7-day archive

Closing a tab does not delete its captures — they move into a 7-day rolling archive in chrome.storage.local. Entries older than 7 days are swept at service-worker startup. You can also manually clear the archive (sidepanel → recents → Clear archive) or reset everything (Settings → Advanced → Reset everything).

5. Sharing — what we send when you click "Send"

  • Webhook destinations (Slack / Discord / Teams): an HTTPS POST to the URL you configured, with the formatted bug report. Payload is the already-masked event from local storage.
  • Web intents (GitHub Issue, mailto): a new browser tab is opened with a pre-filled body. The browser navigates; Hindsight does not POST anything itself.
  • Replay bundle download: a single .html file is written to your computer via chrome.downloads. Nothing is uploaded.
  • ZIP export: same — local file write only.
  • Copy to clipboard: the report is placed on your system clipboard. Nothing leaves until you paste.

A confirmation dialog ("Privacy preview modal") shows the exact event count, redaction summary, and destination identity before every send.

6. Replay requests

The "↻ Replay this request" button re-fires a captured request from the extension's own context. The fetch explicitly uses credentials: 'omit' and skips any header whose stored value is ***MASKED***. Replayed auth-bound requests typically return 401 / 403 — that's expected, the result panel labels it.

7. Permissions — why we ask for each

Permission Reason
storage Read and write the local capture buffer.
unlimitedStorage Allow buffers larger than 5 MB on heavy-browsing days.
activeTab Inspect the tab the user is actively looking at.
scripting Inject the content scripts that observe fetch / XMLHttpRequest / console.
sidePanel Open the primary inspection UI.
notifications (opt-in) Failure-cascade desktop notifications. Requested only when enabled.
downloads (opt-in) Save replay-bundle HTML and ZIP exports.
tabs (opt-in) Read tab metadata for sidepanel + popup display.
webNavigation (opt-in) Detect SPA route changes and reload events.
<all_urls> (opt-in) Run the content script on user pages so request interception works.

8. Telemetry — none

Hindsight collects no usage data, no error reports, no diagnostic pings. No Sentry, no Google Analytics, no Mixpanel, no Amplitude, no Segment. Zero fetch() calls to any Hindsight-owned domain. If this ever changes, it will require explicit opt-in via a Settings checkbox before any data leaves your device, and this policy will be updated first.

9. Cookies — none

Hindsight does not set cookies. It does not store cookies on any external site. It reads the Cookie request header of pages you visit only to mask its value before persisting the network event locally.

10. Third-party services

The extension itself ships with no third-party services bundled. At runtime, optional features touch URLs you configure (your webhook, your mailto: URI). Hindsight does not share these with anyone; they live in your local storage.

11. Children's privacy

Hindsight is a developer tool. It is not marketed to children and does not knowingly capture data from children.

12. Changes to this policy

Material changes will be reflected in this document and called out in the extension's CHANGELOG.

13. Contact

Hindsight is open source. Bug reports, privacy questions, and vulnerability disclosures should go to the project's GitHub repository.
